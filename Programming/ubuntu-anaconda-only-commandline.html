なぜにコマンドラインOnly?

AnacondaをUbuntuにインストールする方法を紹介したコンテンツは世の中に腐るほどあります.

  • Anaconda を利用した Python のインストール (Ubuntu Linux)
  • AnacondaをUbuntu16.04にインストール
  • LinuxにPython3とAnacondaをインストール

ってか,普通にAnacondaのダウンロードページからlinuxのインストールスクリプトをダウンロードボタンを押してDLして あとコマンドラインでスクリプトを実行すればいいだけです.

しかし,世の中にはGUI環境がどうしても使えないってことがよくあるんじゃぁ. あるいはLinuxとかいうギークなツールでGUIとか使いたくないんじゃぁ.みたいな人も稀におります.

私も,研究室にあるssh only(VNCとかその他GUI使えない系)なサーバにAnacondaを入れなきゃいけなかったので,その時に考えた方法を紹介します. ってか,それ以外にもAWSとかのクラウドGPU環境での機械学習環境をフルスクラッチで作成するとかいうときにも使えると思います. まぁAWSとかならAnaconda入ったインスタンスとかあると思うけどさ.

AnacondaをUbuntuの真っ黒な画面のみで入れる方法

まず,curlコマンドは使えるものとします.

wgetでもいいですが私はcurlおじさんなのでcurlでお願いします. なければ

sudo apt-get install curl

などで適宜curlを入れてねん.

ちなみにcurlとか,wgetとかは賢明な読者諸君はすでにご存知だと思うが,そうでない読者のために一言で言うと, 真っ黒画面でweb(http)のやり取りするやつよん.

さて,フォルダを汚したくないので,適当にmkdir anaconda_installationとか言う感じでフォルダを作っておいて, cd anaconda_installationでディレクトリを移動しておきます.

ここからが肝です.

curl -O https://repo.anaconda.com/archive/Anaconda3-5.2.0-Linux-x86_64.sh

で,Anacondaのインストールコマンドをcurlコマンドで(真っ黒画面で)ダウンロードします. -Oは英語のオーのオー文字だよん。数字のゼロではないです。

なお,将来的にバージョンが変わった時のことを考えて補足します.

AnacondaのLinux向けインストーラダウンロードサイトのDownloadボタンのリンクを curl -Oの後に書けば,フォルダ内に,インストーラのコマンドファイルがダウンロードされるって事です.

それだけ.

後は,上で紹介したようなサイトと同様に,インストーラのコマンドファイルにchmod +x Anaconda3-5.2.0-Linux-x86_64.shで実行権限を与えた後, ./Anaconda3-5.2.0-Linux-x86_64.shで実行して,なんか言われるがままに操作していけばおしまいです.

まとめ

ここまでの手順をまとめます.

sudo apt-get install curl; # curlがない場合
mkdir anaconda_installation; # インストールのためのフォルダ作成
curl -O https://repo.anaconda.com/archive/Anaconda3-5.2.0-Linux-x86_64.sh;
chmod +x https://repo.anaconda.com/archive/Anaconda3-5.2.0-Linux-x86_64.sh;
./Anaconda3-5.2.0-Linux-x86_64.sh;

後は,利用規約の説明をEnter連打で進めて,規約にyes. anacondaのインストール場所を聞かれるので,ここは空白でEnter.

Anaconda3 will now be installed into this location:
/home/{Username}/anaconda3

  - Press ENTER to confirm the location
  - Press CTRL-C to abort the installation
  - Or specify a different location below

[/home/{Username}/anaconda3] >>>

パスを追加してもいいか聞かれるので,ここもyesしてEnter.

Do you wish the installer to prepend the Anaconda3 install location
to PATH in your /home/ユーザ名/.bashrc ? [yes|no]
[no] >>> yes

VS Codeインストールしていいか聞かれるから,ここはNo! Noと言える日本人の私であった.

別解

とにかくLinuxのインストーラshellファイルを真っ黒画面onlyなサーバ上に入れればいいんだから, 適当なPCでLinux用ファイルをGUIからダウンロードしてftpで送るなり,USBメモリ使って移すなりすればいいと言えばいい.

ただし,インストーラファイルはでかいんじゃぁ(約500MB).
